```
당신은 Dual-Track UI + Logic TDD 전문가입니다.

우리는 이미 Magic Square(4x4) 시나리오를 작성했습니다.
지금 단계는 구현이 아니라,
시나리오를 “실패하는 테스트(RED)”로 변환하는 것입니다.

중요
- 구현 코드를 작성하지 마십시오.
- GREEN 단계로 가지 마십시오.
- REFACTOR 단계로 가지 마십시오.
- 오직 RED 테스트만 작성하십시오.
- 모든 테스트는 현재 구현이 없기 때문에 실패해야 합니다.

------------------------------------------------------------
프로젝트 조건
------------------------------------------------------------

입력:
- 4x4 int\\\[]\\\[]
- 0은 빈칸
- 정확히 2개의 0
- 값 범위 1\\\~16
- 중복 금지 (0 제외)

출력:
- int\\\[6]
- 좌표는 1-index
- \\\[r1, c1, n1, r2, c2, n2]

마방진 상수:
- 4x4 → 34

------------------------------------------------------------
Dual-Track RED 단계 구조
------------------------------------------------------------

UI RED        Logic RED
   ↓              ↓
(둘 다 실패 상태)

------------------------------------------------------------
TRACK A — UI / Boundary RED
------------------------------------------------------------

다음을 실패 테스트로 변환하십시오:

1. 4x4가 아닌 입력은 예외 발생
2. 빈칸이 2개가 아니면 예외 발생
3. 값 범위 위반 시 예외 발생
4. 중복 숫자 존재 시 예외 발생
5. 반환 배열 길이는 6이어야 한다
6. 반환 좌표는 1-index여야 한다

각 테스트에 대해:
- 테스트 이름
- Given (입력)
- When (호출)
- Then (기대)
- 이 테스트가 보호하는 계약 설명

------------------------------------------------------------
TRACK B — Logic RED
------------------------------------------------------------

다음을 실패 테스트로 작성하십시오:

1. find\\\_blank\\\_coords()
   - row-major 순서로 정확히 2개 반환

2. find\\\_not\\\_exist\\\_nums()
   - 누락 숫자 2개 반환
   - 오름차순

3. is\\\_magic\\\_square()
   - 모든 행 합 동일
   - 모든 열 합 동일
   - 두 대각선 합 동일
   - 34와 일치

4. solution()
   - small → first blank 시도
   - 실패 시 reverse 시도
   - 6-length 반환
   - 1-index 좌표

각 테스트는:
- 실패 상태임을 명확히 설명
- 어떤 Invariant를 검증하는지 명시

------------------------------------------------------------
출력 형식
------------------------------------------------------------

# UI RED Tests
- Test ID
- Scenario
- Invariant 보호 내용

# Logic RED Tests
- Test ID
- Scenario
- Invariant 보호 내용

------------------------------------------------------------

구현 금지
GREEN 단계 금지

참고해서 테스트 케이스 생성해줘 오직 RED 테스트 설계만 수행하십시오

## UI RED Tests (Track A)

| Test ID | Scenario (Given → When → Then) | Invariant / 계약 |
|--------|--------------------------------|------------------|
| **BOUND-MS-UI-001** | 3×3·비직사각 4행 → `solution` → `ValueError` | **FR-1.1** 격자 4×4; 잘못된 형태는 유스케이스 진입 차단 |
| **BOUND-MS-UI-002** | 빈칸 0·1·3개 → `ValueError` | **FR-1.2** 빈칸 정확히 2개 |
| **BOUND-MS-UI-003** | 채워진 칸에 17, 빈칸 2곳 유지 → `ValueError` | **FR-1.4** 1~16 (0은 빈칸만) |
| **BOUND-MS-UI-004** | 0 제외 중복 → `ValueError` | **FR-1.3** 중복 없음 |
| **BOUND-MS-UI-005** | Level 4 유효 부분 격자 → `len(result)==6` | **FR-5.3** 반환 `[r1,c1,n1,r2,c2,n2]` |
| **BOUND-MS-UI-006** | 동일 입력 → 좌표 1~4, 값 1~16·서로 다름 | **FR-5.3** 1-index 좌표·누락 두 수 |
| **BOUND-MS-UI-007** | `None`·문자 셀 → `TypeError` | **§10** 입력 타입 계약 |

진입점: `boundary.magic_square.solution` (GREEN에서 ECB에 맞게 구현).

## Logic RED Tests (Track B)

| Test ID | Scenario | Invariant / 계약 |
|--------|----------|------------------|
| **LOGIC-MS-L-001** | Gherkin 부분 격자 → `[(2,2),(3,3)]` (0-index, row-major) | **FR-2.1~2.3** 빈칸=0, 순서, 2개 |
| **LOGIC-MS-L-002** | 동일 격자 → `[1, 6]` 오름차순 | **FR-3.1~3.2** 누락 두 수·정렬 |
| **LOGIC-MS-L-003** | 알려진 완성 마방진 `True`·깨진 행 `False` | **FR-4, INV-02, INV-03** 10선·합 34 |
| **LOGIC-MS-L-004** | `len==6`, 1-index 범위, 기대 `[3,3,6,4,4,1]`, 채운 뒤 `is_magic_square` `True` | **FR-5.1~5.3** 시도 순서·6원소·1-index |
| **LOGIC-MS-L-005** | 완성 격자 스왑 후 `False` | **INV-02** 대각 포함 10선 판정 |

모듈: `entity.magic_square_ops` — `find_blank_coords`, `find_not_exist_nums`, `is_magic_square`, `solution`.
