## 1. 작업 개요

1. **Dual-Track UI + Logic TDD** 관점에서, 사용자가 정리한 **Boundary(관측 계약)** 항목과 **Entity(순수 규칙)** 항목을 각각 pytest로 옮겼다.
2. **구현 코드는 작성하지 않았다.** `boundary.magic_square` 모듈과 `entity.magic_square_ops` 모듈은 아직 없으며, 테스트 실행 시 **모듈 부재로 실패(RED)** 하도록 두었다.
3. **입력·출력 계약**은 사용자 합의와 PRD를 따른다. 부분 격자는 4×4 정수, `0`은 빈칸 **정확히 2개**, 채워진 값은 1~16·중복 없음. `solution` 성공 시 반환은 **길이 6의 `int` 리스트**, 좌표는 **1-index**, 형식 `[r1, c1, n1, r2, c2, n2]`. 매직 합은 **34**.
4. Logic 트랙은 **테스트 케이스 명세서(스프레드시트)** 및 Level 4 **Gherkin Given**과 맞는 고정 격자(알려진 완성 마방진·부분 격자)를 사용한다.
5. 검증 실행: 저장소 루트에서 `python -m pytest tests -q` 시 **19건 실패**가 나오는 것이 현재 기대 상태(RED)이다.
