## 1. 공용 상수·Entity (Track B)

### 1.1 `constants.MATRIX_SIZE`

- 루트 모듈 [`constants.py`](../constants.py)에 `MATRIX_SIZE = 4` (**Final**)를 두어, 격자 차원을 **한곳**에서 참조한다.
- `pyproject.toml`의 `[tool.setuptools]`에 **`py-modules = ["constants"]`** 로 포함해 editable 설치 시 import 가능하게 했다.

### 1.2 `entity.magic_square_ops`

| 함수 | 역할 |
|------|------|
| `find_blank_coords` | 값 `0`인 칸을 **행 우선(0-index)** 으로 나열한다. |
| `find_not_exist_nums` | `1..n²` 중 격자에 없는 수를 **오름차순**으로 반환한다 (`n = MATRIX_SIZE`). |
| `is_magic_square` | **0이 없는 완전 격자**에서 행·열·주·부 대각 **10선**이 모두 매직 합과 일치하는지 판별한다. 매직 합은 `MATRIX_SIZE * (n² + 1) // 2`로 유도한다. |
| `solution` | FR-5와 동일하게 **빈칸 순서 고정** 후, 누락 두 수를 **오름차순 `(a,b)`로 먼저 배치**, 실패 시 **`(b,a)`**를 시도한다. 반환은 **1-index** 길이 6 리스트. |

- RED 테스트: [`tests/entity/test_magic_square_logic_red.py`](../tests/entity/test_magic_square_logic_red.py) — **LOGIC-MS-L-001 ~ L-005** 전부 이 모듈을 기준으로 통과한다.

## 2. Boundary (Track A) 보강

- **`validate(grid)`** 공개 API: 내부 `_validate_grid`와 동일하게 **FR-1 계열 검증**만 수행하고, 통과 시 `list[list[int]]`를 반환한다.
- **GUI 계약**: `validate` → `solution` 순서로 호출할 수 있게 하여, Screen에서 **검증 실패와 풀이 실패**를 단계적으로 처리할 수 있다.
- `_validate_grid`의 **행·열 개수** 및 해당 함수 내 전체 셀 순회에는 **`MATRIX_SIZE`** 를 사용한다. (나머지 헬퍼 일부는 여전히 리터럴 `4`를 쓰는 구간이 있을 수 있다.)

## 3. 패키징 (`pyproject.toml`)

- **`[build-system]`**: `setuptools>=69`, `build-backend = setuptools.build_meta`.
- **패키지 명시**: `boundary`, `entity`, `magicsquare`, `magicsquare.screen` + `constants` 단일 모듈.  
  → 다중 탑레벨 폴더(`prompt`, `Report` 등)가 **실수로 패키지에 포함되지 않도록** flat-layout 자동 탐색 오류를 피한다.
- **선택 의존성**
  - `dev`: `pytest>=7.0`
  - `gui`: `PyQt6>=6.6.0`

## 4. GUI (Screen 전용, PyQt6)

### 4.1 마방진 퍼즐 창

- 경로: [`magicsquare/screen/app.py`](../magicsquare/screen/app.py)
- **진입**: 프로젝트 루트에서 **`python -m magicsquare`** (공식 1경로, README에 기재).
- **동작**: `MATRIX_SIZE`×`MATRIX_SIZE` `QSpinBox`(0~16), 「풀기」→ **`validate` → `solution`**, 오류는 `QMessageBox`, 성공 시 라벨에 길이 6 결과. 기본 격자는 Gherkin식 **빈칸 2개** 부분 격자.
- **제약**: PyQt import는 **`magicsquare.screen`** 아래에만 둔다. `entity`·`constants`·`boundary`는 Qt를 import 하지 않는다.

### 4.2 pytest 출력 창 (테스트 GUI)

- 경로: [`magicsquare/screen/test_runner.py`](../magicsquare/screen/test_runner.py)
- **진입**: **`python -m magicsquare --tests`** (또는 첫 인자 `tests`).
- **동작**: 저장소 루트를 working directory로 두고 **`sys.executable -m pytest tests -v --color=no`** 를 `QProcess`로 실행, stdout/stderr를 `QTextEdit`에 스트리밍한다. **pytest는 `[dev]`** 에 있으므로, 테스트 GUI까지 쓰려면 **`pip install -e ".[dev,gui]"`** 권장.
- **주의**: venv에 **`[gui]`만** 있고 PyQt6가 없으면 `__main__`에서 안내 메시지 후 종료한다. (과거 백그라운드 실행에서 exit code 1로 관측된 케이스.)
