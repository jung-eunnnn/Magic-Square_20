## 1. 구현 요약

### 1.1 진입점

- **`boundary.magic_square.solution(grid)`**  
  - 반환: 길이 6의 `list[int]` — `[r1, c1, n1, r2, c2, n2]` (행·열 **1-index**, 두 빈칸에 넣을 수 **1~16**).
  - PRD·RED 계약: 잘못된 형태·빈칸 수·범위·중복·타입은 **경계에서 거부**한다.

### 1.2 검증(경계)

| 구분 | 동작 |
|------|------|
| `None` / 비정수 셀 | `TypeError` (셀은 `type(x) is int`로 검사해 `bool` 혼입 방지) |
| 4×4 아님, 비직사각 행 | `ValueError` |
| 빈칸(`0`) 개수 ≠ 2 | `ValueError` |
| 0이 아닌 값이 1~16 밖 | `ValueError` |
| 0 제외 값 중복 | `ValueError` |

### 1.3 풀이 로직

- 빈칸 좌표는 **행 우선**으로 수집한다.
- 격자에 없는 두 수는 **오름차순** `[a, b]`로 둔 뒤, **먼저 첫 빈칸·둘째 빈칸에 `(a, b)`**를 넣어 4×4 매직(행·열·두 대각선 합 34) 여부를 검사하고, 실패 시 **`(b, a)`**를 시도한다. (Entity RED의 FR-5·Gherkin Given과 동일한 순서.)
- 두 배치 모두 매직이 아니면 `ValueError`를 올린다. (현재 RED 픽스처는 첫 시도 실패 후 역순으로 성공한다.)
