## 1. 변경 배경

- 기존에는 테스트 실행 GUI(`--tests`)와 커버리지 확인이 분리되어 있었고, 커버리지는 터미널/HTML 리포트 중심이었다.
- 사용자 요청으로 **HTML 대신 GUI 표 형태**의 커버리지 확인 기능을 추가했다.
- 이후 사용 중 발견된 문제(창 미표시, `--cov` 인자 오류, 테스트 탭 접근 불편)를 반영해 UI/실행 경로를 재정비했다.

---

## 2. 구현 내용

### 2.1 의존성

- [`pyproject.toml`](../pyproject.toml)
  - `[project.optional-dependencies].dev`에 `pytest-cov>=4.0` 추가
  - `[project.optional-dependencies].gui`에 `pytest`, `pytest-cov` 포함
  - 목적: GUI 실행 환경에서 `--cov` 옵션 인식 실패 방지

### 2.2 엔트리포인트

- [`magicsquare/__main__.py`](../magicsquare/__main__.py)
  - `python -m magicsquare --tests`
  - `python -m magicsquare --coverage`
  - 두 경로 모두 최종적으로 **통합 러너 창**을 사용하도록 정리

### 2.3 커버리지 GUI 추가

- 신규 파일 [`magicsquare/screen/coverage_gui.py`](../magicsquare/screen/coverage_gui.py)
  - `QProcess`로 `python -m pytest tests -q --cov=... --cov-report=json:<temp>` 실행
  - coverage JSON을 읽어 파일별 통계를 `QTableWidget`에 표시
  - 합계(구문/실행/누락/커버%)를 라벨로 제공
  - 행 선택 시 미커버 라인 번호 표시
  - 임시 JSON 파일 생성 후 파싱 완료 시 삭제

### 2.4 통합 탭 UI

- [`magicsquare/screen/test_runner.py`](../magicsquare/screen/test_runner.py)
  - `UnifiedRunnerWindow` 추가
  - `QTabWidget`으로 **테스트 실행 / 커버리지** 2개 탭 제공
  - `run_unified_runner_app(initial_tab=...)`로 초기 탭 선택
  - `--tests`/`--coverage` 호출 시 같은 창을 재사용하는 UX로 통일

## 3. 이슈 및 수정

### 3.1 커버리지 창이 뜨지 않거나 즉시 닫힘

- 원인: 보조 창 객체 참조가 유지되지 않아 가비지 컬렉션으로 소멸 가능
- 조치: 창 인스턴스를 리스트로 보관하고 `destroyed` 시 정리하도록 변경

### 3.2 `unrecognized arguments: --cov=...`

- 원인: 실행 중인 Python 환경에 `pytest-cov` 미설치
- 조치:
  - 의존성에 `pytest-cov` 반영
  - GUI 실행 전 `pytest_cov` 존재 여부를 점검해 친절한 안내 메시지 표시

### 3.3 커버리지 모드에서 테스트 화면 접근 혼란

- 원인: 커버리지 전용 창과 테스트 창이 분리되어 사용자가 "탭이 사라짐"으로 인지
- 조치: 단일 통합 창으로 구조 변경 (`QTabWidget`)

## 4. 실행 방법 (최신)

프로젝트 루트, 가상환경 활성화 후:

```powershell
Set-Location "c:\DEV\Magic-Square_20"
python -m pip install -e ".[gui]"
```

통합 GUI 실행:

```powershell
python -m magicsquare --tests
```

또는

```powershell
python -m magicsquare --coverage
```

- 두 명령 모두 동일한 통합 창을 띄우며, 초기 선택 탭만 다르다.
